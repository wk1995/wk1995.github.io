# 模块 07：个人股票系统 UI 与功能设计

## 1. 模块目标

为个人 AI 股票学习与工具平台设计一套可落地的 UI 方案，明确页面结构、功能范围、核心工作流和交互规则。

这个系统的 UI 不是营销网站，而是一个长期使用的个人工作台。设计重点是：

- 新手能看懂
- 信息密度足够
- 风险提示清楚
- 操作路径短
- 每个判断都能追溯原因
- 不鼓励冲动交易

## 2. 产品定位

```text
个人股票学习 + 研究 + 监控 + 复盘 + 风控工作台
```

系统不做：

- AI 荐股神器
- 自动下单工具
- 纯行情看盘软件
- 公开社区产品
- 高风险交易工具

系统要做：

- 帮用户学习股市知识
- 帮用户管理自选股
- 帮用户理解股票状态
- 帮用户记录交易计划
- 帮用户导入成交记录并生成日报
- 帮用户复盘错误
- 帮用户控制风险

## 3. 设计原则

### 3.1 后台工具风格

页面应偏冷静、克制、实用，类似个人金融工作台。

建议：

- 左侧导航 + 顶部状态栏
- 表格、图表、列表为主
- 卡片只用于明确分组，不做装饰堆叠
- 重点数字突出，但不过度视觉刺激
- 风险、止损、亏损信息要明显

### 3.2 新手友好

每个复杂指标都应该能解释。

示例：

```text
RSI: 42
解释: 动量偏弱，但已经脱离超卖区。
注意: RSI 不能单独作为买入理由。
```

### 3.3 行动前先检查

涉及交易计划的页面，优先展示：

- 买入理由
- 止损位
- 目标位
- 盈亏比
- 仓位
- 是否计划内

不要把“买入”作为视觉中心。第一版只做记录和分析，不做真实交易按钮。

### 3.4 信号必须可解释

所有信号都要展示：

- 触发时间
- 信号等级
- 触发原因
- 关键价位
- 失效条件
- 后续复盘入口

## 5. 页面功能清单

### 5.1 Dashboard 首页

目标：每天打开网站先知道今天该关注什么。

支持功能：

- 今日市场状态
- 自选股信号概览
- 最新反弹/突破/破位信号
- 今日热点新闻
- 命中自选股或持仓的消息
- 当前持仓风险摘要
- 今日交易日报入口
- 今日学习任务
- 明日计划提醒
- 数据导入提醒

核心区域：

```text
顶部状态栏
  - 日期
  - 市场状态
  - 数据更新时间
  - 风险等级

左侧主区域
  - 自选股状态表
  - 最新信号
  - 今日待办

右侧辅助区域
  - 风控摘要
  - 热点新闻
  - AI 今日提示
  - 学习进度
```

首页不要放太多解释性长文，优先展示可行动信息。

### 5.2 Watchlist 自选股

目标：管理关注股票，看到每只股票的当前状态。

支持功能：

- 添加股票
- 删除股票
- 设置关注理由
- 设置支撑位、压力位、失效位
- 添加标签
- 查看最新价格
- 查看涨跌幅
- 查看信号状态
- 查看风险状态
- 排序和筛选

建议字段：

```text
股票
价格
涨跌幅
信号状态
关注理由
关键支撑
关键压力
失效位
标签
更新时间
操作
```

筛选项：

- 全部
- 有信号
- 反弹观察
- 反弹确认
- 破位风险
- 持仓中
- 计划内

### 5.3 Stock Detail 个股详情

目标：把一只股票的价格、信号、基本面、计划和复盘集中到一个页面。

支持功能：

- K线图
- 成交量
- 技术指标
- 支撑/压力/止损线标记
- 最新信号解释
- AI 公司摘要
- 新闻公告摘要
- 相关新闻热度
- 交易计划
- 历史交易记录
- 复盘记录

页面布局：

```text
顶部
  - 股票名称
  - 代码
  - 当前价格
  - 涨跌幅
  - 信号状态

主区域
  - K线图
  - 成交量
  - 指标面板

右侧
  - 信号解释
  - 关键价位
  - 交易计划
  - AI 风险提示

底部
  - 新闻公告
  - 热点消息
  - 财报摘要
  - 历史交易和复盘
```

### 5.4 News 热点消息

目标：聚合市场热点，帮助用户快速理解当天最重要的消息和风险。

支持功能：

- 全市场热点榜
- 财经热点榜
- 板块热点榜
- 自选股相关新闻
- 持仓相关新闻
- 新闻来源列表
- 热度分
- AI 摘要
- 关联股票和板块
- 风险提醒
- 原文链接

建议字段：

```text
排名
标题
热度分
来源数
发布时间
分类
情绪
影响级别
关联板块
关联股票
操作
```

详情页展示：

```text
一句话摘要
核心事实
来源列表
相关股票
相关板块
可能影响
不确定性
风险提示
明日是否继续观察
```

### 5.5 Signals 信号中心

目标：集中查看所有股票触发的技术信号。

支持功能：

- 信号列表
- 信号详情
- 信号过滤
- 信号分数
- 触发原因
- 关键价位
- 失效条件
- 后续表现追踪
- 复盘入口

信号类型：

```text
反弹观察
反弹预警
反弹确认
突破确认
破位风险
止损提醒
```

信号详情应该展示：

```text
股票
时间
周期
信号类型
分数
触发条件
关键价位
失效条件
风险收益比
后续 5/15/30 分钟表现
```

### 5.6 Research AI 研究助手

目标：用 AI 帮用户整理信息，而不是替用户做决定。

支持功能：

- 输入股票代码生成公司摘要
- 粘贴公告生成摘要
- 粘贴新闻生成影响分析
- 粘贴财报生成财务摘要
- 生成风险清单
- 生成待验证问题
- 检查交易计划

界面结构：

```text
左侧
  - 输入区
  - 分析类型
  - 资料来源

中间
  - AI 结构化输出

右侧
  - 保存到个股档案
  - 生成交易计划草稿
  - 风险提示
```

AI 输出必须区分：

- 事实
- 推断
- 风险
- 待验证问题

### 5.7 Learn 学习

目标：让新手按路径学习，不是零散浏览。

支持功能：

- 学习路线
- 知识卡片
- 小测题
- 个人笔记
- 今日学习任务
- 错题和误区
- 与真实股票案例关联

页面结构：

```text
学习首页
  - 当前进度
  - 今日任务
  - 推荐复习

知识卡片详情
  - 概念解释
  - 常见误区
  - 示例
  - 相关股票案例
  - 小测题
```

### 5.8 Strategy 策略回测

目标：验证规则，不靠感觉。

支持功能：

- 策略列表
- 策略配置
- 参数设置
- 历史回放
- 回测报告
- 信号历史表现
- 策略对比

核心指标：

```text
信号次数
胜率
盈亏比
最大回撤
平均持有时间
最大连续亏损
计划内交易表现
```

第一版可以只展示回测结果，不需要复杂可视化。

### 5.9 Journal 交易日志

目标：记录每次交易想法和执行结果。

支持功能：

- 新建交易计划
- 记录买入理由
- 记录止损位
- 记录目标位
- 记录仓位
- 记录执行结果
- 记录情绪状态
- AI 复盘

交易计划表单必须包含：

```text
股票
方向
买入理由
入场参考
止损位
目标位
计划仓位
最大亏损
是否已有信号
是否符合风控
```

### 5.10 Reports 交易日报

目标：每天收盘后生成交易总结。

支持功能：

- 上传成交记录
- 上传持仓快照
- 生成今日概览
- 展示交易明细
- 展示持仓变化
- 计算盈亏归因
- 引用当日热点新闻
- 生成纪律评分
- AI 生成日报总结
- 生成明日计划
- 保存日报

日报页面结构：

```text
顶部
  - 日期
  - 总盈亏
  - 交易次数
  - 纪律评分
  - 风险等级

主区域
  - 今日交易明细
  - 持仓变化
  - 盈亏归因

右侧
  - AI 总结
  - 热点影响
  - 今日问题
  - 明日计划
```

### 5.11 Risk 风控

目标：让用户知道自己最多可能亏多少。

支持功能：

- 总仓位
- 单只股票仓位
- 行业集中度
- 最大潜在亏损
- 止损覆盖率
- 无止损交易提醒
- 连续亏损提醒
- 风控规则检查

风控提示等级：

```text
正常
注意
警告
严重
```

严重风险要放在首页和个股页显著位置。

### 5.12 Import 数据导入

目标：把同花顺或券商导出的交易数据导入系统。

支持功能：

- 上传 CSV
- 上传 Excel
- 字段映射
- 数据预览
- 重复记录检查
- 异常值检查
- 导入确认
- 导入历史

导入流程：

```text
选择文件
  -> 识别模板
  -> 字段映射
  -> 数据校验
  -> 预览
  -> 确认导入
  -> 生成日报或更新持仓
```

第一版优先支持手动导入，不做自动登录同花顺。

### 5.13 Settings 设置

目标：配置数据、AI、风控和通知。

支持功能：

- 市场偏好
- 数据源配置
- AI 输出风格
- 风控规则
- 通知方式
- 默认指标参数
- 导入模板管理
- 本地数据备份

## 6. 核心工作流

### 6.1 新手学习流

```text
Dashboard
  -> Learn 今日任务
  -> 知识卡片
  -> 小测题
  -> 保存学习笔记
```

### 6.2 个股研究流

```text
Watchlist
  -> Stock Detail
  -> News 查看相关热点
  -> Research AI
  -> 保存公司摘要和风险点
  -> 创建交易计划
```

### 6.3 热点消息流

```text
Dashboard
  -> News 热点榜
  -> 查看来源和 AI 摘要
  -> 打开关联股票
  -> 检查是否影响自选股、持仓或交易计划
```

### 6.4 信号观察流

```text
Signals
  -> 查看信号原因
  -> 打开个股详情
  -> 检查关键价位和失效条件
  -> 创建或更新交易计划
```

### 6.5 交易复盘流

```text
Import
  -> 上传成交记录
  -> 生成 Reports 日报
  -> 引用热点新闻解释市场环境
  -> AI 总结问题
  -> 写入 Journal
  -> 更新 Risk 看板
```

### 6.6 策略验证流

```text
Strategy
  -> 配置规则
  -> 运行回测
  -> 查看结果
  -> 保存策略版本
  -> 对比真实交易表现
```

## 7. 状态和视觉标识

### 7.1 信号状态

```text
NO_SIGNAL: 无信号
WATCH: 观察
REBOUND_WARNING: 预警
REBOUND_CONFIRMED: 确认
BREAKOUT_CONFIRMED: 突破
BREAKDOWN_RISK: 破位
INVALIDATED: 失效
```

### 7.2 风险状态

```text
NORMAL: 正常
NOTICE: 注意
WARNING: 警告
CRITICAL: 严重
```

### 7.3 交易计划状态

```text
DRAFT: 草稿
READY: 可观察
TRIGGERED: 已触发
EXECUTED: 已执行
CLOSED: 已结束
REVIEWED: 已复盘
```

### 7.4 新闻影响状态

```text
POSITIVE: 偏利好
NEGATIVE: 偏利空
NEUTRAL: 中性
UNCERTAIN: 不确定
RISK: 风险消息
```

## 8. 组件建议

基础组件：

- 左侧导航
- 顶部状态栏
- 股票搜索框
- 数据表格
- 筛选器
- 标签
- 状态徽标
- 指标面板
- K线图容器
- AI 输出面板
- 交易计划表单
- 风险提示条
- 热点新闻列表
- 新闻摘要面板
- 导入向导
- 日报卡片
- 复盘抽屉

交互组件：

- Tabs：个股详情中切换概览、技术、研究、交易、复盘
- Drawer：从信号列表打开信号详情
- Drawer：从热点列表打开新闻事件详情
- Modal：确认导入、删除计划、保存日报
- Toast：导入成功、保存成功、数据异常
- Tooltip：解释指标和评分
- Empty State：没有自选股、没有交易记录、没有日报

## 9. 桌面和移动适配

桌面端是主场景，适合深度分析。

桌面端重点：

- 左侧导航常驻
- 表格和图表并排
- 个股详情支持右侧信息栏
- 日报和风控支持多列布局

移动端只做轻量查看。

移动端重点：

- 查看自选股
- 查看最新信号和热点新闻
- 查看日报摘要
- 查看风险提醒
- 记录简单交易想法

移动端暂不建议做复杂回测和大量字段导入。

## 10. MVP 页面范围

第一版建议只做 7 个页面：

```text
Dashboard
Watchlist

News
Journal
Reports
Import
```

第一版必须支持：

- 手动添加自选股
- 查看个股基础信息

- 展示反弹信号状态
- 创建交易计划
- 上传成交记录

## 12. 验收标准

- 用户能在首页知道今天重点关注什么
- 用户能管理自选股并看到信号状态
- 用户能打开个股详情查看价格、指标、信号和计划
- 用户能查看热点新闻并理解关联股票、板块和风险
- 用户能创建完整交易计划
- 用户能导入成交记录生成日报
- 用户能看到纪律评分和主要问题
- 用户能看到当前持仓风险
- 所有 AI 输出都能保存和复盘
- 系统不诱导用户直接交易
