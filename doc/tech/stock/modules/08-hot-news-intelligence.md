# 模块 08：热点消息系统

## 1. 模块目标

构建一个热点消息模块，从公开新闻源、财经媒体、公告源和数据 API 中捕获热度靠前的消息，进行聚合、去重、热度排序、AI 总结，并关联到股票、板块、策略信号和交易日报。

这个模块的目标不是让用户追消息交易，而是帮助用户：

- 快速知道今天市场在关注什么
- 识别影响自选股和持仓的消息
- 区分事实、情绪和推断
- 发现板块轮动和主题热度
- 在交易前检查是否有突发风险
- 在日报中解释当日盈亏和市场环境

## 2. 功能边界

系统要做：

- 聚合多个新闻源
- 捕获热点新闻
- 做新闻去重和合并
- 计算热度分
- 提取关键词、股票、行业和板块
- AI 生成简短摘要
- 标注利好、利空、中性或不确定
- 推送和自选股、持仓相关的消息
- 写入交易日报和个股档案

系统不做：

- 未授权抓取付费内容
- 绕过媒体反爬或登录限制
- 把新闻摘要当成买卖建议
- 用单条新闻直接触发自动交易
- 编造新闻来源和影响结论

## 3. 数据来源

### 3.1 来源类型

建议采用“可配置来源注册表”，不要把某个媒体写死在代码里。

来源分层：

```text
官方 API
  -> 新闻搜索 API
  -> 金融数据 API
  -> 公告和交易所 API

公开 RSS
  -> 财经媒体 RSS
  -> 证券媒体 RSS
  -> 科技媒体 RSS

公开网页
  -> 只在授权和合规前提下采集标题、摘要、链接
  -> 不抓取付费正文
  -> 不绕过访问限制

手动导入
  -> 用户粘贴新闻
  -> 用户导入新闻链接
```

### 3.2 可选数据源

国际新闻和英文新闻：

- NewsAPI：提供 top headlines 和 everything 等新闻接口
- GDELT：全球新闻事件和媒体数据，适合做大范围监控
- 官方媒体 RSS：例如 Reuters、AP、CNBC、BBC、Financial Times 等公开 RSS 或公开频道

中文财经新闻：

- 交易所公告：上交所、深交所、港交所等公告源
- 财经媒体公开频道：财联社、证券时报、中国证券报、上海证券报、证券日报、第一财经、每日经济新闻、界面新闻、澎湃新闻、东方财富、同花顺财经等
- 数据服务商：Tushare、iFinD、Wind、Choice、聚源等

说明：

- 不同来源的授权、可访问性和使用条款差异很大
- 第一版优先用公开 RSS、官方 API、手动导入和用户可授权的数据源
- 如果要长期运行，推荐接入合规的数据服务商，而不是依赖网页抓取

### 3.3 参考接口

可用于原型验证的公开接口方向：

- NewsAPI：适合获取 top headlines 和关键词新闻
- GDELT：适合监控全球新闻事件、媒体覆盖和跨语言新闻
- 媒体公开 RSS：适合低成本获取标题、摘要和原文链接
- 交易所公告源：适合获取上市公司公告和监管信息

注意：

- 搜索和新闻 API 的授权政策会变化，接入前必须重新确认最新条款
- 如果新闻来自付费数据服务，需要确认是否允许二次展示摘要
- 如果只是个人本地使用，也应保留原文链接和来源，不复制长篇正文

## 4. 核心流程

```text

  -> 标准化字段
  -> 去重和合并
```


## 6. 热度计算

热度分建议使用 100 分制。

```text
热度分 = 来源权重 + 时间新鲜度 + 重复报道数 + 传播速度 + 市场反应 + 用户相关性
```

建议权重：

```text
来源权重：20分
时间新鲜度：20分
重复报道数：20分
传播速度：15分
市场反应：15分
用户相关性：10分
```

解释：

- 来源权重：权威媒体、交易所公告、官方渠道权重更高
- 时间新鲜度：越新的消息权重越高
- 重复报道数：多个独立来源报道同一事件，热度上升
- 传播速度：短时间内多源出现，热度上升
- 市场反应：相关股票或板块异动，热度上升
- 用户相关性：命中自选股、持仓或关注行业，权重上升

## 7. 去重和事件合并

新闻去重不能只按标题完全匹配。

推荐规则：

```text
标题相似度
关键词重合度
发布时间窗口
关联股票和板块
新闻来源数量
正文摘要相似度
```

合并逻辑：

```text
如果两条新闻在 2 小时内发布
且标题相似度较高
且关键词重合度较高
则合并为同一新闻事件
```

同一事件下保留：

- 最早来源
- 最新来源
- 来源列表
- 不同媒体标题
- AI 合并摘要
- 热度变化曲线

## 8. 股票和板块关联

关联方式：

```text
股票代码匹配
公司简称匹配
公司全称匹配
行业关键词匹配
板块关键词匹配
概念关键词匹配
用户自选股标签匹配
```

需要注意：

- 公司简称可能有歧义
- 同名词可能不是股票名称
- AI 关联结果必须允许人工修正
- 关联强度要打分，不要只给 true / false

关联结果：

```json
{
  "symbol": "300750",
  "name": "宁德时代",
  "relation_type": "company_mentioned",
  "confidence": 0.92,
  "reason": "新闻标题和正文摘要直接出现公司名称"
}
```

## 9. AI 摘要和分类

AI 对每个聚合事件输出：

```text
一句话摘要
核心事实
可能影响
相关股票和板块
利好/利空/中性/不确定
短期情绪影响
中长期影响是否需要验证
风险提示
待验证问题
```

AI 输出示例：

```json
{
  "one_line": "多家媒体报道新能源产业政策落地，相关板块热度上升。",
  "facts": [
    "消息来源包括多家财经媒体",
    "事件发生在今日早盘",
    "相关板块出现成交放大"
  ],
  "inference": [
    "短线可能提升市场风险偏好",
    "实际业绩影响仍需后续数据验证"
  ],
  "sentiment": "positive",
  "impact_horizon": "short_term",
  "risk": [
    "消息落地后可能出现高开低走",
    "不能仅凭热点追涨"
  ]
}
```


## 13. MVP 实现

第一版建议只做：

```text
配置 3-5 个公开新闻源
  -> 定时获取标题和摘要
  -> 按关键词去重
  -> 计算简化热度分
  -> AI 生成短摘要
  -> 展示 Top 10 热点
  -> 标记命中自选股和持仓的新闻
```

第一版页面：

- News 热点页
- Dashboard 热点新闻卡片
- Stock Detail 相关新闻区
- Reports 日报新闻摘要

第一版不做：

- 大规模爬虫
- 付费正文抓取
- 自动交易触发
- 复杂舆情模型
- 社交媒体情绪采集

## 15. 风险和限制

- 新闻源使用必须遵守对应网站和 API 的条款
- 热点不等于机会，很多热点是短线情绪
- 新闻摘要可能遗漏关键细节，需要保留原文链接
- AI 可能误判利好利空，必须标注不确定性
- 同一事件可能被多个媒体重复报道，需要去重
- 部分新闻会在盘后才体现，不能简单和即时涨跌绑定
- 谣言、误传和标题党需要通过来源权重和多源验证过滤

## 16. 验收标准

- 能配置新闻源
- 能定时获取新闻标题、摘要、来源和链接
- 能去重并合并同一事件
- 能按热度输出 Top N 新闻
- 能生成 AI 摘要和风险提示
- 能关联自选股、持仓、行业和板块
- 能在首页、个股页和日报中展示相关新闻
- 能保留原文链接和来源列表
- 不绕过登录、付费墙或反爬限制
