# 模块 06：交易日报与 AI 总结

## 1. 模块目标

结合交易记录、持仓快照、行情数据、策略信号和用户笔记，自动生成每日交易日报。

日报的核心目标不是证明今天赚了多少钱，而是回答：

- 今天做了哪些交易
- 这些交易是否符合计划
- 盈亏来自哪里
- 哪些错误重复出现
- 当前持仓风险是否可控
- 明天应该重点观察什么

## 2. 输入数据

### 2.1 交易记录

第一阶段建议支持手动录入或上传 CSV / Excel。

来源可以是：

- 同花顺交易软件导出的成交记录
- 券商交易系统导出的成交记录
- 手动录入的模拟交易记录
- 网站内部保存的交易计划和模拟成交

说明：

- 普通同花顺客户端通常不适合作为网站直接查询真实账户的公开 API 来源
- 初期应优先采用“导出文件 -> 上传解析 -> 本地分析”的只读方式
- 后续如果券商提供官方量化接口，再接入只读查询持仓、成交、资金

交易记录字段：

```json
{
  "trade_id": "20260625-001",
  "trade_date": "2026-06-25",
  "trade_time": "10:31:22",
  "symbol": "600519",
  "name": "贵州茅台",
  "market": "CN",
  "side": "BUY",
  "price": 1680.5,
  "quantity": 100,
  "amount": 168050,
  "fee": 35.2,
  "source": "ths_export"
}
```

### 2.2 持仓快照

持仓快照用于计算仓位、浮动盈亏和风险暴露。

```json
{
  "snapshot_date": "2026-06-25",
  "symbol": "600519",
  "name": "贵州茅台",
  "quantity": 100,
  "available_quantity": 100,
  "cost_price": 1650.0,
  "last_price": 1680.5,
  "market_value": 168050,
  "unrealized_pnl": 3050,
  "position_ratio": 0.18
}
```

### 2.3 行情和信号

用于解释交易环境。

需要数据：

- 当日开高低收
- 当日涨跌幅
- 成交量和成交额
- 指数和板块表现
- 当日触发的反弹、突破、破位信号
- 用户自选股关注价位

### 2.4 用户交易计划

日报需要对比“计划”和“实际执行”。

```json
{
  "plan_id": "plan-20260625-001",
  "symbol": "600519",
  "direction": "long",
  "reason": "回踩支撑后反弹确认",
  "entry_reference": 1680,
  "stop_loss": 1648,
  "target": 1745,
  "max_loss_ratio": 0.01,
  "created_at": "2026-06-25T09:10:00+08:00"
}
```

## 3. 日报生成流程

```text
1. 导入当日成交记录
2. 导入或生成收盘持仓快照
3. 拉取或导入当日行情数据
4. 关联当天交易计划和策略信号
5. 清洗和标准化字段
6. 计算盈亏、仓位、费用和交易频率
7. 检查是否违反交易规则
8. 生成结构化日报
9. AI 生成自然语言总结
10. 用户确认并保存日报
```

## 4. 日报结构

每日交易日报建议包含以下部分。

### 4.1 今日概览

```text
日期：
交易市场：
交易次数：
买入金额：
卖出金额：
手续费：
已实现盈亏：
浮动盈亏变化：
账户总盈亏：
最大单笔风险：
交易纪律评分：
```

### 4.2 市场环境

记录当天大盘和板块环境：

```text
指数表现：
市场情绪：
强势板块：
弱势板块：
今日是否适合积极交易：
```

### 4.3 今日交易明细

每笔交易记录：

```text
股票：
方向：
成交价格：
成交数量：
成交金额：
交易理由：
是否符合计划：
关联信号：
当前结果：
问题标签：
```

### 4.4 持仓变化

```text
新增持仓：
减仓股票：
清仓股票：
仓位变化：
行业集中度变化：
浮动盈亏变化：
```

### 4.5 盈亏归因

把盈亏拆成几个来源：

```text
策略内交易盈亏
计划外交易盈亏
持仓波动盈亏
手续费和滑点
大盘/板块影响
个股事件影响
```

### 4.6 纪律评分

建议 100 分制。

```text
交易前有计划：20分
设置止损位：20分
仓位符合规则：20分
没有追涨杀跌：15分
按信号执行：15分
交易后完成复盘：10分
```

扣分示例：

```text
无计划交易：-20
无止损交易：-20
仓位过重：-15
连续亏损后继续加仓：-20
冲动追涨：-10
跌破失效位未处理：-20
```

### 4.7 AI 总结

AI 总结应包含：

- 今天做得好的地方
- 今天主要问题
- 盈亏背后的原因
- 是否遵守交易计划
- 是否存在情绪交易
- 明天最重要的风险提醒
- 明天观察清单

## 7. 字段映射

不同券商或同花顺导出的文件字段名可能不同，需要做字段映射。

示例映射：

```json
{
  "成交日期": "trade_date",
  "成交时间": "trade_time",
  "证券代码": "symbol",
  "证券名称": "name",
  "买卖方向": "side",
  "成交价格": "price",
  "成交数量": "quantity",
  "成交金额": "amount",
  "手续费": "fee"
}
```

导入时需要检查：

- 日期格式是否正确
- 股票代码是否标准化
- 买卖方向是否能识别
- 数量、价格、金额是否为数字
- 是否存在重复成交记录
- 手续费是否缺失

## 8. 关键指标

日报至少计算：

- 当日交易次数
- 买入金额
- 卖出金额
- 手续费
- 已实现盈亏
- 浮动盈亏变化
- 总仓位
- 单只股票最大仓位
- 行业集中度
- 计划内交易比例
- 无止损交易次数
- 冲动交易次数
- 纪律评分

进阶指标：

- R 倍数
- 平均盈利
- 平均亏损
- 当日胜率
- 盈亏比
- 最大单笔亏损
- 最大潜在亏损
- 策略内交易盈亏 vs 计划外交易盈亏

## 9. AI 提示词模板

```text
你是我的交易复盘助手。

请基于以下数据生成今日交易日报：
1. 成交记录
2. 持仓快照
3. 今日市场环境
4. 今日策略信号
5. 我的交易计划

要求：
- 不给出直接买卖建议
- 区分事实、推断和待验证问题
- 重点指出是否违反交易纪律
- 总结今天最重要的 3 个问题
- 给出明天交易前需要检查的清单
- 语言简洁，不要安慰，不要夸大
```

## 10. AI 输出约束

AI 不能：

- 编造不存在的成交
- 编造不存在的收益
- 直接建议明天买哪只股票
- 把短期盈利解释成稳定能力
- 忽略止损、仓位和风险

AI 必须：

- 基于导入数据总结
- 明确数据缺失项
- 标记计划外交易
- 标记无止损交易
- 标记仓位异常
- 输出明日风险提醒

## 11. MVP 实现

第一版只做最小闭环：

```text
手动上传成交 CSV
  -> 字段映射
  -> 数据预览
  -> 生成交易统计
  -> AI 生成日报
  -> 保存 Markdown / JSON
```

第一版暂不做：

- 自动登录同花顺
- 自动下单
- 自动读取真实资金账号
- 复杂归因模型
- 多账户合并

- 能导入当天交易记录
- 能识别买入、卖出、费用和成交金额
- 能生成当日交易统计
- 能对比交易计划和实际成交
- 能标记计划外交易和无止损交易
- 能生成 AI 日报总结
- 能保存日报并用于后续周报、月报统计
