# 模块 03：行情监控与信号系统

## 1. 模块目标

把网站变成实时或准实时的股票观察工具。

核心不是简单展示价格，而是回答：

- 当前股票处于什么状态
- 是否触发观察信号
- 信号为什么触发
- 关键价位在哪里
- 信号什么时候失效

## 4. 信号类型

第一阶段建议实现 4 类信号：

```text
反弹观察
反弹确认
突破确认
破位风险
```

### 4.1 反弹观察

用于提示可能止跌。

条件示例：

```text
最近一段时间存在下跌背景
价格不再继续创新低
RSI 从超卖区回升
出现长下影线
成交量放大但价格没有继续破位
```

### 4.2 反弹确认

参考已有文档：

- `../rebound-signal-monitor.md`

核心结构：

```text
L0 = 第一次低点
H1 = 反抽高点
L1 = 回踩低点

L1 > L0
价格突破 H1
突破时成交量放大
```

### 4.3 突破确认

用于识别强势股突破。

条件示例：

```text
价格突破最近 N 根K线高点
价格突破关键压力位
成交量大于均量 1.5 倍
突破后没有快速跌回压力位下方
```

### 4.4 破位风险

用于控制亏损。

条件示例：

```text
跌破关键支撑
跌破止损位
放量长阴
跌破 EMA20 或 EMA60
大盘或板块同步走弱
```

## 7. 实现阶段

### P0：手动数据版本

- 手动维护自选股
- 手动输入关键价位
- 手动记录信号观察

### P1：行情数据版本

- 接入行情数据
- 自动计算指标
- 自动生成信号

### P2：AI 解释版本

- AI 根据信号原因生成解释
- AI 提醒风险点
- AI 生成复盘问题

## 8. 验收标准

- 能展示自选股列表
- 能展示个股详情
- 能计算基础指标
- 能输出至少一种信号
- 信号必须有原因、关键位和失效条件
