# 模块 02：AI 股票研究助手

## 1. 模块目标

用 AI 帮助用户整理信息，而不是让 AI 直接做买卖决定。

研究助手应该输出结构化结果：

- 公司做什么
- 业务靠什么赚钱
- 最近发生了什么
- 财务表现如何
- 当前风险是什么
- 需要继续验证什么

## 2. 核心功能

### 2.1 公司概览

输入股票代码，生成：

- 公司名称
- 所属市场
- 所属行业
- 主营业务
- 核心产品
- 收入来源
- 主要竞争对手

### 2.2 财报摘要

AI 对财报做结构化总结：

- 收入变化
- 利润变化
- 毛利率变化
- 现金流变化
- 负债变化
- 管理层展望
- 需要注意的异常项

### 2.3 新闻和公告摘要

AI 读取新闻或公告后输出：

- 事件是什么
- 可能利好还是利空
- 影响短期还是长期
- 影响收入、利润、估值还是情绪
- 不确定性是什么

### 2.4 投资假设生成

帮助用户把想法变成可验证的假设。

示例：

```text
假设：
这只股票未来上涨的核心原因是利润增长恢复。

需要验证：
1. 下一季度收入是否增长
2. 毛利率是否改善
3. 行业需求是否回暖
4. 股价是否站回关键均线

失效条件：
1. 利润继续下滑
2. 现金流恶化
3. 跌破关键支撑位
```

## 3. 页面设计

```text
AI Research
  - 输入股票代码
  - 选择分析类型
  - 上传或粘贴资料
  - 生成结构化分析
  - 保存到个股档案

分析类型
  - 公司概览
  - 财报摘要
  - 新闻公告解读
  - 风险清单
  - 交易计划检查
  - 长期投资假设
```

## 4. 输出格式

```json
{
  "symbol": "AAPL",
  "analysis_type": "company_summary",
  "summary": "公司主营消费电子、服务和软件生态。",
  "bullish_points": [
    "服务收入占比提升",
    "现金流稳定"
  ],
  "bearish_points": [
    "硬件增长放缓",
    "估值可能偏高"
  ],
  "unknowns": [
    "下一代产品需求",
    "监管风险"
  ],
  "follow_up_questions": [
    "最近一个季度服务收入增长多少？",
    "当前估值和历史区间相比处于什么位置？"
  ]
}
```

## 5. AI 约束

AI 输出必须遵守：

- 不直接输出“必涨”“必买”
- 必须列出不确定性
- 必须区分事实、推断和观点
- 财务结论尽量关联原始数据
- 无数据时明确标注“需要补充数据”
- 每次分析都生成风险提示

## 6. 交易计划检查

用户输入计划后，AI 做完整性检查。

检查项：

- 买入理由是否清晰
- 是否有止损位
- 是否有目标位
- 盈亏比是否合理
- 是否和大盘环境冲突
- 是否因为情绪冲动
- 是否有财报、公告、政策等事件风险

输出示例：

```text
计划完整度：65/100

缺失项：
- 没有明确止损位
- 没有说明如果跌破支撑如何处理
- 没有确认财报发布时间

建议：
- 先补充失效条件
- 仓位控制在试探级别
- 不要在重大事件前满仓
```

## 7. 验收标准

- 能输入股票代码生成基础研究摘要
- 能粘贴财报、公告或新闻生成结构化摘要
- 能输出风险点和待验证问题
- 能检查交易计划是否完整
- 能保存分析结果到个股档案
