# 模块 04：策略回测与交易复盘

## 1. 模块目标

帮助用户验证自己的判断，而不是凭感觉交易。

这个模块要解决：

- 规则过去是否有效
- 信号触发后通常怎么走
- 最大亏损可能是多少
- 自己最常犯什么错误
- AI 能否辅助发现复盘盲点

## 2. 策略模板

每个策略都必须结构化保存。

```json
{
  "id": "rebound-v1",
  "name": "反弹确认策略 V1",
  "market": "US",
  "timeframe": "5m",
  "entry_rules": [
    "出现更高低点",
    "突破反抽高点",
    "成交量大于20根均量1.5倍"
  ],
  "exit_rules": [
    "跌破 L1",
    "达到目标位",
    "持有超过指定时间仍未上涨"
  ],
  "risk_rules": [
    "单笔最大亏损不超过账户1%",
    "盈亏比不低于2"
  ]
}
```

## 3. 回测指标

至少统计：

- 信号次数
- 胜率
- 平均盈利
- 平均亏损
- 盈亏比
- 最大回撤
- 最大连续亏损
- 单次最大亏损
- 平均持有时间
- 不同市场环境下表现

## 4. 历史回放

支持按时间重放行情，避免“看未来”。

流程：

```text
选择股票
  -> 选择日期范围
  -> 选择策略
  -> 按时间逐根K线回放
  -> 记录每次信号
  -> 统计信号后表现
```

## 6. AI 复盘

AI 复盘不判断“对错”，而是检查流程。

检查项：

- 买入前是否有计划
- 是否按计划执行
- 是否临时改变止损
- 是否因为情绪追涨
- 是否忽略大盘环境
- 是否仓位过重
- 是否重复犯同类错误

AI 复盘输出：

```text
本次交易主要问题：
1. 买入理由清晰，但止损设置太远
2. 盈亏比不足 2:1
3. 交易前没有确认大盘状态

下次改进：
1. 先计算最大亏损再下单
2. 如果盈亏比不足，不进入确认信号
3. 记录入场前的情绪状态
```

## 7. 页面设计

```text
Strategy
  - 策略列表
  - 策略编辑
  - 参数配置
  - 回测运行
  - 回测报告

Replay
  - 股票选择
  - 时间范围
  - K线回放
  - 信号标记

Journal
  - 交易记录
  - 交易截图
  - 交易理由
  - AI 复盘
  - 统计分析
```

## 8. 验收标准

- 能保存策略规则
- 能记录每次模拟交易
- 能统计交易结果
- 能生成 AI 复盘摘要
- 能知道策略是否经过历史验证
