# 个人 AI 股票学习与工具平台方案

## 1. 背景

目标用户是股市新手，希望通过 AI 提升学习、研究、监控、复盘和风控能力，并把个人网站逐步建设成自己的股票工具平台。

本方案的定位不是“AI 自动荐股”或“AI 保证赚钱”，而是：

- 帮助新手建立正确的投资和交易知识体系
- 把模糊的判断变成可记录、可验证、可复盘的流程
- 用 AI 提升信息整理、策略验证和风险控制效率
- 让网站成为个人长期使用的股票决策辅助工具

## 2. 核心原则

```text
先学习，再模拟，再小资金验证
先风控，再收益
先记录，再优化
先规则，再 AI
先辅助决策，再考虑交易联动
```

必须坚持的底线：

- AI 是研究助理，不是操盘手
- 不用 AI 的单次回答直接买卖股票
- 不做没有止损位的交易
- 不用杠杆、期权、融资融券作为新手起步工具
- 不做无法复盘和无法验证的策略
- 不把短期盈利误判为稳定能力

## 3. 总体目标

把网站建设成 7 类工具的组合：

```text
学习系统
  -> AI 研究助手
  -> 行情监控与信号系统
  -> 热点消息系统
  -> 策略回测与交易复盘
  -> 交易日报与总结
  -> 风控与资产管理
```

最终形态：

- 能学习股票基础知识
- 能维护自选股
- 能查看股票核心信息
- 能监控技术信号，例如反弹、突破、破位
- 能捕获市场热点新闻并关联股票、板块和持仓
- 能让 AI 总结财报、公告、新闻和风险点
- 能记录每一次交易想法和结果
- 能结合交易记录生成每日交易总结
- 能回测自己的规则是否有效
- 能提醒自己控制仓位、止损和集中度

## 6. 技术架构建议

第一阶段可以轻量实现，不急着上复杂后端。

```text
静态网站
  -> 本地 JSON / CSV 数据
  -> 浏览器 LocalStorage
  -> 手动录入自选股和交易日志
  -> 简单指标计算
```

第二阶段增加数据接入：

```text
前端页面
  -> API 服务
  -> 行情数据源
  -> 股票基础数据
  -> 新闻/公告/财报数据
  -> AI 摘要服务
  -> 本地数据库
```

第三阶段增加策略和回测：

```text
历史行情数据
  -> 策略引擎
  -> 回测引擎
  -> 统计分析
  -> 策略报告
  -> 信号复盘
```

## 7. 数据类型

至少需要以下数据：

- 股票基础信息：代码、名称、市场、行业、板块
- 行情数据：开高低收、成交量、成交额
- 指标数据：均线、RSI、MACD、ATR、VWAP
- 财务数据：收入、利润、毛利率、现金流、负债
- 新闻公告：标题、时间、来源、摘要、影响方向
- 热点新闻：来源、发布时间、热度分、关键词、关联股票、AI 摘要
- 自选股数据：关注理由、观察价位、风险点
- 交易日志：买入理由、卖出理由、止损位、结果、复盘
- 交易记录：成交时间、方向、价格、数量、金额、费用
- 持仓快照：持仓数量、成本、市值、浮动盈亏、仓位占比
- 交易日报：当日盈亏、交易次数、纪律评分、错误标签、明日计划
- 策略信号：触发时间、触发条件、信号分数、后续表现

## 8. AI 能力边界

AI 适合做：

- 解释概念
- 总结财报和公告
- 提炼新闻影响
- 聚合热点新闻并生成摘要
- 识别新闻关联的股票、板块和风险
- 生成观察清单
- 检查交易计划是否完整
- 帮助写策略规则
- 帮助复盘交易错误
- 生成每日交易总结
- 识别重复出现的交易问题
- 辅助生成回测报告

AI 不适合做：

- 直接保证买哪只股票能赚钱
- 单独预测明天涨跌
- 替代风险控制
- 替代数据验证
- 替代用户自己的交易决策

## 9. 阶段规划

### P0：学习和记录

目标：

- 建立学习页面
- 建立自选股页面
- 建立交易想法记录
- 建立基础术语和知识卡片

验收标准：

- 能记录自选股
- 能写下关注理由
- 能记录模拟买卖
- 能复盘每一次判断

### P1：行情和指标

目标：

- 接入或导入行情数据
- 展示 K线和成交量
- 计算 EMA、RSI、MACD、ATR
- 标记支撑、压力、趋势状态

验收标准：

- 能看到一只股票的核心技术状态
- 能解释当前上涨、下跌、震荡或反弹观察状态

### P2：AI 研究助手

目标：

- AI 总结公司基本面
- AI 总结公告和新闻
- AI 生成风险清单
- AI 帮助生成交易计划草稿

验收标准：

- 输入股票代码后，能生成结构化研究摘要
- 每个结论尽量关联数据来源或用户输入
- 明确标注不确定性和风险点

### P3：信号监控

目标：

- 实现反弹信号
- 实现突破信号
- 实现破位风险信号
- 输出信号原因和关键价位

验收标准：

- 信号不是黑盒
- 每个信号都有触发条件
- 每个信号都有失效条件
- 每个信号都能后续复盘

### P4：回测、复盘和日报

目标：

- 支持历史回放
- 支持策略规则测试
- 支持交易日志统计
- 支持导入交易记录并生成日报
- 支持 AI 复盘亏损原因

验收标准：

- 能看到策略胜率、盈亏比、最大回撤
- 能看到信号触发后的历史表现
- 能生成每日交易总结和明日计划
- 能发现自己最常犯的错误

### P5：风控和交易联动

目标：

- 建立仓位管理
- 建立风险预算
- 建立止损提醒
- 建立组合风险看板

验收标准：

- 能知道当前最大潜在亏损
- 能知道是否过度集中某只股票或某个行业
- 在考虑真实交易前，必须经过模拟和回测验证

## 10. 推荐先做的最小版本

第一版不要做大而全，建议只做：

```text
自选股
  -> 个股详情
  -> 学习卡片
  -> 交易想法记录
  -> 反弹信号展示
```

最小页面：

- 首页 Dashboard
- 自选股 Watchlist
- 个股详情 Stock Detail
- 学习 Learn
- 复盘 Journal
- 日报 Reports

最小能力：

- 手动添加股票
- 手动记录关注理由
- 展示基础行情
- 展示基础指标
- 使用已有反弹信号方案输出观察状态
- 保存模拟交易记录
- 手动录入或导入成交记录生成日报

## 11. 成功标准

短期成功标准不是赚钱，而是：

- 能稳定学习
- 能少犯低级错误
- 能解释自己的每次判断
- 能知道每次交易最多亏多少
- 能在交易前写下计划
- 能在交易后复盘结果
- 能通过数据验证策略，而不是靠感觉

长期成功标准：

- 有稳定的交易规则
- 有经过回测的策略
- 有可控的回撤
- 有清晰的仓位纪律
- 有持续复盘和迭代机制
